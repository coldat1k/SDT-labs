## Документація по веб-застосунку
**Призначення:** REST API сервіс для обліку обладнання (Simple Inventory). Дозволяє додавати нове обладнання, переглядати повний список та детальну інформацію про конкретну одиницю за її ідентифікатором.

**Запуск та налаштування:**
Усі залежності та налаштування середовища виконуються автоматично за допомогою скрипта розгортання `setup.sh`. Застосунок працює як systemd-сервіс (`mywebapp.service`) під управлінням системного користувача `app`.

## Документація по API-ендпоінтах
API підтримує зміну формату відповіді (HTML або JSON) залежно від заголовку `Accept` у запиті клієнта.

* `GET /` — Кореневий ендпоінт. Повертає список доступних маршрутів (тільки HTML).
* `GET /items` — Виводить список усіх предметів в інвентарі (id, name).
* `POST /items` — Створює новий запис. Очікує параметри `name` та `quantity`.
* `GET /items/<id>` — Виводить детальну інформацію по запису (id, name, quantity, created_at).
* `GET /health/alive` — Перевірка стану застосунку. Завжди повертає HTTP 200 OK.
* `GET /health/ready` — Перевірка підключення до бази даних. Повертає HTTP 200 OK при успішному з'єднанні.

## Документація по розгортанню
* **Базовий образ:** Офіційний образ Ubuntu 22.04 LTS (або 24.04 LTS).
* **Вимоги до ресурсів:** 1 vCPU, 1 GB RAM, 10 GB Disk (мінімальні вимоги).
* **Спеціальні налаштування OS:** Не потрібні.
* **Доступ:** Вхід виконується через SSH під стандартним користувачем віртуальної машини (наприклад, `ubuntu`), який після розгортання блокується. Подальший вхід можливий через створених користувачів `student` або `teacher`.

**Інструкція з розгортання:**
1. Клонувати репозиторій на віртуальну машину.
2. Перейти в директорію проєкту: `cd SDT-labs`.
3. Надати права на виконання скрипту: `chmod +x setup.sh`.
4. Запустити скрипт з правами root: `sudo ./setup.sh`.

## Інструкція з тестування розгорнутої системи
